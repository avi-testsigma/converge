02

The Problem

AI coding tools generate code fast. But nobody verifies it automatically. The human is still the only quality gate.

🤖

Code Agents Are Blind

Claude Code, Codex, Cursor generate code — but don't know if it actually works for the user's intent. Output is "pure luck" without execution.

👥

Orchestrators Multiply the Problem

Conductor, Claude Squad run 5-10 agents in parallel. Great for throughput — but the review bottleneck grows linearly. More agents = more to verify manually.

⏳

Review Doesn't Scale

Reading every line of AI-generated code, manually testing, mentally comparing against intent, writing feedback — this kills the productivity gain from parallel agents.

Writing code is cheap now. But good code — correct, tested, matching intent — is still expensive. Every orchestrator optimizes for generating more code faster. Nobody optimizes for generating more confidence faster.

— Inspired by Simon Willison, "Agentic Engineering Patterns"
03

Two Insights

What if the AI generated both the code AND its verification criteria — from the same requirement?

🧪

Tests Are Executable Specifications

Generate tests FROM the requirement (not from the code). Tests become an automated verification layer between AI and human. The human reviews test adequacy — "does this capture what I want?" — rather than code correctness.

Tests are shorter, more readable, and directly tied to intent. This is TDD at the orchestration level.

🎨

Visual Prototypes Are Reviewable Specs

A mockup is faster to approve than a written spec. Generate a visual prototype first, then derive both code and tests from the approved visual.

The mockup becomes the visual contract. Screenshot comparison against it is automated verification of design intent.

04

How Converge Works

Seven phases from requirement to merged, verified code.

1

Envision

Describe a feature. Converge generates visual mockups via Gemini Nano Banana Pro. Pick one, iterate ("make the button blue"), approve.

2

Decompose & Ground

The approved vision is broken into parallel tasks. A grounding agent explores your codebase — framework, conventions, existing tests — and passes context to all agents.

3

Dual-Track Generate

Code agents and test agents work simultaneously in isolated git worktrees. Code agents implement. Test agents generate .sigma behavior tree acceptance tests from the requirement — not from the code.

4

Red Check

Generated tests run against the current codebase (before code changes). Every test must fail. If a test passes without new code, it's vacuous — it tests nothing. Flagged and regenerated.

5

Converge

Code is merged. Existing tests run first (regression check). Then BT tests run. Failures produce structured diagnostics — which step failed, screenshots, DOM state. Fix agents are dispatched. Loop until all tests pass and the visual matches the mockup.

6

Review

Read the change walkthrough — a narrative summary of what was built. See test pass rate, visual similarity, regression status. Review generated tests in the visual editor. Everything that reaches you has already passed automated verification.

7

Merge

One click. Changes merged to your branch. Generated tests kept as regression suite. Worktrees cleaned up.
